Foveated Transformer for 3D Occupancy Prediction

Alexandra Lagutova, Bryan Gotti, Loïc Misenta, Louise Montlahuc
Project Code Dataset Website Code

Foveated Transformer We propose a fovea-like, distance-aware voxelization within an existing framework, adapting spatial resolution across encoder and decoder to reduce computation while preserving detail in instance-critical regions.

Abstract

Current 3D perception models rely on uniform voxel grids that process all regions at the same resolution, leading to massive computational inefficiencies since much of 3D space is empty. Overcoming this bottleneck is critical for real-time applications like autonomous driving, where strict latency constraints demand faster inference without sacrificing vital spatial details.

Inspired by biological foveated vision, we propose a distance-aware, non-uniform tokenization method for 3D voxel transformers. This approach dynamically allocates high-resolution tokens to critical, nearby regions while using coarser representations for distant or empty space, successfully reducing the overall token count while preserving essential geometric detail.

I. Introduction

Our project introduces a foveated transformer framework for 3D semantic occupancy prediction. In this task, a 3D scene is represented as a voxel grid, and the model predicts whether each voxel is occupied and which semantic class it belongs to. This representation is particularly important for autonomous driving, where accurate and efficient 3D scene understanding is required before downstream planning and decision-making.

3D occupancy prediction with voxels.

A key limitation of standard voxel-based models is that they process the entire 3D volume at a uniform resolution, even though much of the space is empty or less relevant. Inspired by biological foveated vision, where the retina provides high acuity near the center of gaze and lower resolution in the periphery, we investigate whether a 3D occupancy model can allocate more computation to safety-critical regions while using coarser representations elsewhere.

Acuity vs. distance from fovea.

Building on VoxDet, our work explores foveated mechanisms for improving the efficiency of 3D occupancy prediction. Specifically, we study how distance-aware weighting and non-uniform token allocation can reduce unnecessary computation while preserving detail in regions that are most important for scene understanding.

II. Related Works

In 2D vision, foveated transformers such as FoveaTercite iteratively select regions of interest and process them at higher resolution while maintaining a coarse peripheral representation. Similarly, adaptive tokenization methods such as Adaptive Token Samplingcite learn to prune uninformative tokens dynamically, enabling input-dependent computation. While effective, these 2D methods focus on token selection rather than redefining the underlying spatial representation.

In 3D perception, related ideas appear through sparse and hierarchical voxel representations. For instance, VoxFormercite adopts a sparse-to-dense strategy, where only visible voxels are explicitly represented before a transformer infers the full scene. Similarly, geometry-aware voxel transformerscite improve efficiency by leveraging structured attention over voxel grids, implicitly focusing computation on meaningful regions. These methods reduce computational cost by limiting the number of processed voxels; however, they still rely on a fixed voxelization scheme within each stage, and the decoder typically operates on a dense grid.

In contrast, this work introduces a fovea-like adaptive tokenization directly into the voxel representation within the VoxDet frameworkcite. We redefine the spatial structure itself using non-uniform sampling.

Overall, while prior work focuses on token pruning or sparse attention, our approach embeds adaptivity directly into the voxelization process, enabling a more efficient and geometrically structured 3D detection framework.

III. Methodology

Motivation and Baseline Architecture

Current voxel-based perception models typically process 3D space with a uniform resolution, treating all regions with equal importance. Inspired by human foveated vision, where perception is highly detailed around a focus area, our project investigates whether a 3D occupancy model can dynamically allocate more capacity to safety-relevant or closer regions, while remaining compatible with standard uniform voxel grid evaluations.

To establish our baseline, we utilized the VoxDetcite architecture. Originally, VoxDet frames occupancy prediction as an instance-centric problem featuring both a semantic branch and an instance-level regression branch (which predicts a 4D offset field). To obtain a clean baseline for our foveated modifications, we simplified the architecture by entirely removing the instance-level regression branch, allowing us to focus exclusively on semantic occupancy. Our experimental pipeline was built around the SemanticKITTI and KITTI Vision Benchmark datasets, processed and executed on the EPFL cluster. In this simplified baseline, the training objective consists of three terms: a geometric scaling loss and a semantic scaling loss (which act as global regularizers), and a standard voxel-wise cross-entropy (CE) loss that supervises each voxel independently.

Adaptive loss

Starting from this baseline, our first major modification was replacing the standard voxel-wise CE loss with an Adaptive Forward-Distance Weighted Loss. Because the loss function does not natively receive camera calibration matrices or depth maps, we use the voxel index along the x-axis (the forward direction of the ego-vehicle) as a proxy for distance. As voxel tokens pass through the image-view transformer and the occupancy encoder, the prediction head (OccHead) outputs per-voxel class logits. Before reducing the loss, we apply a dynamically computed multiplicative weight to each voxel's cross-entropy value. Closer voxels receive higher weights, effectively shifting the gradient magnitudes so the optimizer focuses more heavily on updating parameters for near-field, safety-critical regions. To maintain training stability, this weight is clamped to a range of [0.5,1.5]. While our framework supports exponential and inverse distance weighting (as well as up-weighting for dynamic classes, boundaries, and uncertainty), computational constraints on the cluster led us to focus our experiments on a linear forward-distance weighting scheme. Notably, we only apply this foveated weighting to the cross-entropy term; the global scaling losses remain unweighted to preserve overall scene consistency.

The Foveated Loss Function: Instead of a standard mean cross-entropy, our foveated loss applies a spatial weight $W(x_v)$ to each valid voxel $v$: $$ \mathcal{L}_{\text{fovea}} = \frac{1}{N_{valid}} \sum_{v \in \mathcal{V}} W(x_v) \cdot \mathcal{L}_{\text{CE}}(v) $$ Where the weight function $W(x_v)$ is based on the voxel's forward coordinate $x_v$, scaled by a configuration factor $\lambda$, and clamped to prevent gradient explosion: $$ W(x_v) = \text{Clamp}\Big( f_{\text{linear}}(x_v, \lambda), \; 0.5, \; 1.5 \Big) $$

Foveated tokenization

Inspired by FoveaTer (ICLR 2023), foveated tokenization is applied at two distinct stages of the pipeline: first at the level of 3D feature extraction in the voxel backbone, and second at the level of cross-attention queries in the BEV lifting head.

Part 1 — Foveated Local Aggregator (backbone stage)

The FoveatedLocalAggregator replaces the standard dense 3D backbone with a two-stream design that shares weights between both streams. Given the full voxel feature volume of shape $(B, C, V_H, V_W, V_Z)$, two parallel forward passes are computed:

  • Foveal stream: A spatial crop centred at the fixation point is extracted at full resolution and passed through the shared backbone and FPN neck. This preserves fine-grained spatial detail in the region of interest.
  • Global stream: The entire volume is first average-pooled by a factor of 2 in every spatial dimension, halving the resolution, and then processed through the same backbone and FPN neck. This captures wide-field context at a fraction of the cost.

The outputs are merged by upsampling the global feature map back to full resolution via nearest-neighbour interpolation, then overwriting the foveal region with the high-resolution foveal features. The result is a complete $(B, C, V_H, V_W, V_Z)$ feature map in which the central foveal zone retains full-resolution features while the periphery carries coarser, pooled features. Because the backbone and neck weights are shared across both streams, no additional parameters are introduced. This design reduces backbone FLOPs from approximately 685 GFLOPs (dense) to approximately 151 GFLOPs, a 4.5× reduction.

Part 2 — Foveated cross-attention tokenization (transformer stage)

The second application occurs inside VoxFormerHeadCrossAttention, which lifts image features into the voxel grid via deformable cross-attention. Without foveation, every non-masked voxel in the grid is used as a separate query token, which is expensive for large volumes. With foveated tokenization, the set of active voxel queries is instead partitioned into three concentric zones defined by their $L_\infty$ (Chebyshev) distance from the same fixation point used in Part 1:

  • Foveal zone (distance ≤ foveal_radius): every voxel is kept as an individual query token at full resolution.
  • Mid zone (distance between foveal_radius and mid_radius): voxels are grouped into non-overlapping spatial blocks of size mid_stride³; each block is pooled into a single representative token.
  • Peripheral zone (distance > mid_radius): voxels are grouped into blocks of size peripheral_stride³ and similarly reduced to one token per block.

After cross-attention, peripheral and mid-zone block features are scattered back to all member voxels within each block. This reduces the number of transformer queries proportionally to the pooling strides, lowering the attention cost significantly while concentrating computational budget on the foveal region where spatial precision matters most.

Together, the two parts form a coherent non-uniform representation: Part 1 ensures the backbone produces richer features in the foveal zone, and Part 2 ensures the cross-attention head devotes proportionally more query capacity to that same zone.
